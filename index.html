ORA NFS-e Automático

Interface de acesso para operar o sistema sem Render, usando processamento local.

Verificando agente local Abrir local
Agente local necessário

O GitHub mostra a interface. O processamento fiscal continua local.

Para manter as funcionalidades completas — certificados A1, consulta NFS-e Nacional/ADN, XMLs, relatórios, retenções e conferência Excel — o sistema precisa rodar um agente no computador. O GitHub Pages não executa Python nem acessa certificados por segurança.

Tentar abrir agente
Sem Render Não precisa contratar servidor para operar. Cada usuário roda o agente localmente.
Dados protegidos Certificados, senhas, XMLs e base fiscal ficam fora do GitHub.
Interface pública O link do GitHub Pages vira a porta de entrada para o sistema.

Como operar

Etapa 01 Baixe ou clone o repositório. Use git clone ou o botão Code → Download ZIP no GitHub.
Etapa 02 Execute o agente local. No Windows, dê dois cliques em iniciar_github_pages_windows.bat.
Etapa 03 Volte para este link. Assim que o agente responder em 127.0.0.1:8000, esta página carrega o sistema completo.
Segurança Não envie dados fiscais ao GitHub. A pasta data/, certificados e XMLs reais continuam ignorados pelo Git.
